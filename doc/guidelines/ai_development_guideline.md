# 生成AI向け開発ドキュメント運用方針

## 基本方針

開発ガイドラインは、生成AIに常時すべて読ませるものではなく、
タスクの種類に応じて必要なドキュメントのみを参照させる。

生成AIへの実装指示では、以下を明確にする。

1. 今回の作業目的
2. 対象範囲
3. 必ず参照するドキュメント
4. 必要に応じて参照するドキュメント
5. 今回は参照不要なドキュメント
6. 優先順位が衝突した場合の判断基準

## ドキュメントの階層

### Level 1: 常時参照ルール

全タスクで必ず守る最小限のルール。

例:
- プロジェクト概要
- アーキテクチャ原則
- 命名規則
- ディレクトリ構成
- 禁止事項

### Level 2: 作業種別ごとのルール

作業内容に応じて参照する。

例:
- 画面実装ガイドライン
- API実装ガイドライン
- DB設計ガイドライン
- テスト実装ガイドライン
- エラーハンドリング方針

### Level 3: 機能固有ドキュメント

対象Featureや画面に関係する場合のみ参照する。

例:
- Feature定義
- 画面詳細
- 画面遷移図
- テーブル定義
- 業務ルール

## 生成AIへの指示方針

生成AIには、毎回すべてのドキュメントを読ませない。

今回の作業に必要なドキュメントだけを指定し、
参照順序と優先順位を明示する。

## 優先順位

ルールが衝突した場合は、以下の順で優先する。

1. ユーザーからの今回の明示指示
2. 対象Featureの仕様
3. 対象画面・API・DBの詳細設計
4. 作業種別ごとのガイドライン
5. プロジェクト共通ガイドライン

## 注意事項

生成AIは大量の情報を入力すると、すべてを均等に扱えるわけではない。

そのため、ドキュメント管理では「網羅性」だけでなく、
「今回読ませる範囲を絞ること」を品質管理の一部として扱う。
